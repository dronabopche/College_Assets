Que:	






Implement an age calculator using HTML and JavaScript. The user's DOB should be collected through a text box.








Sol: 












<!DOCTYPE html>
<html>
<head>
    <title>Age Calculator</title>

    <script>
        function calculateAge() {

            var dob = document.getElementById("dob").value;

            // Convert DOB into Date object
            var birthDate = new Date(dob);

            var today = new Date();

            var age = today.getFullYear() - birthDate.getFullYear();

            // Check if birthday has occurred this year
            var monthDifference = today.getMonth() - birthDate.getMonth();

            if (
                monthDifference < 0 ||
                (monthDifference === 0 && today.getDate() < birthDate.getDate())
            ) {
                age--;
            }

            // Display result
            document.getElementById("result").innerHTML =
                "Your Age is: " + age + " years";
        }
    </script>
</head>

<body>

    <h2>Age Calculator</h2>

    Enter Date of Birth:
    <input type="date" id="dob">

    <br><br>

    <button onclick="calculateAge()">
        Calculate Age
    </button>

    <h3 id="result"></h3>

</body>
</html>

Que:	




Design a registration page using HTML that includes field for name, rol number, phone number and a submit button. When the user click the button, validate the form using JavaScript. 
The validation rules are as follows: All fields are mandatory, The name field should contain only alphabets and whitespaces, Roll number must start with 23BTechIT or 24BTechIT, followed by 2 digits, Phone number should contain only digits and must be exactly 10 digits.







Sol:	









<!DOCTYPE html>
<html>
<head>
    <title>Student Registration Form</title>

    <script>

        function validateForm()
        {
            var name = document.getElementById("name").value.trim();
            var roll = document.getElementById("roll").value.trim();
            var phone = document.getElementById("phone").value.trim();

            if(name == "" || roll == "" || phone == "")
            {
                alert("All fields are mandatory");
                return false;
            }

            var namePattern = /^[A-Za-z ]+$/;

            if(!namePattern.test(name))
            {
                alert("Name should contain only alphabets and whitespaces");
                return false;
            }

            var rollPattern = /^(23BTechIT|24BTechIT)[0-9]{2}$/;

            if(!rollPattern.test(roll))
            {
                alert("Invalid Roll Number");
                return false;
            }

            var phonePattern = /^[0-9]{10}$/;

            if(!phonePattern.test(phone))
            {
                alert("Phone Number must contain exactly 10 digits");
                return false;
            }

            alert("Registration Successful");
            return true;
        }

    </script>

</head>

<body>

    <h2>Student Registration Form</h2>

    <form onsubmit="return validateForm()">

        Name:
        <input type="text" id="name">
        <br><br>

        Roll Number:
        <input type="text" id="roll">
        <br><br>

        Phone Number:
        <input type="text" id="phone">
        <br><br>

        <input type="submit" value="Submit">

    </form>

</body>
</html>

StudentServlet.java








import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class StudentServlet extends HttpServlet {

    protected void doPost(
            HttpServletRequest request,
            HttpServletResponse response)
            throws ServletException, IOException {

        response.setContentType("text/html");

        PrintWriter out = response.getWriter();

        String name = request.getParameter("name");

        String roll = request.getParameter("roll");

        String gender = request.getParameter("gender");

        String[] skills = request.getParameterValues("skills");

        out.println("<html><body>");

        out.println("<h2>Submitted Information</h2>");

        out.println("<b>Name:</b> " + name + "<br><br>");

        out.println("<b>Roll Number:</b> " + roll + "<br><br>");

        out.println("<b>Gender:</b> " + gender + "<br><br>");

        out.println("<b>Programming Skills:</b><br>");

        if(skills != null) {

            for(String skill : skills) {

                out.println(skill + "<br>");

            }

        }

        out.println("</body></html>");
    }
}

possible issue that can arise:

1. The Port 8080 Conflict (GlassFish vs. Oracle XE)
    The Problem: Oracle Database XE automatically claims port 8080 for its internal HTTP server. GlassFish Server also defaults to port 8080 for its web applications. When you try to run your JSP/Servlet project, NetBeans will throw a "Port already in use" or server startup error.

The Resolution:
    You need to change the GlassFish port.

    1. In NetBeans, go to the Services tab.

    2. Expand Servers, right-click GlassFish Server, and select Properties.

    3.Look for the HTTP port setting (usually under a domain configuration) and change it from 8080 to 8081 or 9090.

    4. Restart GlassFish.



2. JDK Version Mismatches (UnsupportedClassVersionError)
The Problem:
    If you practice at home on a newer Java version (like JDK 17 or 21) and bring your project files to the lab on a pendrive, the older GlassFish server (which likely runs on JDK 8) will crash with java.lang.UnsupportedClassVersionError. This means the .class files were compiled by a newer Java version and cannot be understood by the older lab system.

The Resolution:
    Best Practice: Bring your raw .java, .jsp, and .html source code to the lab, not the compiled NetBeans project folder. Create a fresh NetBeans project on the lab computer and paste your code in.

    Alternative: In your home NetBeans, right-click your project -> Properties -> Sources -> change the Source/Binary Format to match the lab's JDK (likely JDK 8 or 1.8).




3. Database Credentials & Connection Strings
The Problem:
    In your display.jsp and other database files, the code uses system and your_password. Lab computers often have standard, pre-set passwords that differ from what you use at home. Furthermore, the ojdbc14.jar mentioned in the instructions is extremely old (built for Java 1.4).

The Resolution:
    Passwords: Ask the lab assistant for the Oracle XE password. Common defaults in educational labs are system / oracle, system / manager, or system / tiger.

    The JAR File: If NetBeans throws a ClassNotFoundException for oracle.jdbc.driver.OracleDriver, double-check that ojdbc14.jar is actually added to the library in the lab's NetBeans exactly as outlined in your instructions. If the lab has JDK 8, you might need to find ojdbc6.jar or ojdbc8.jar instead, as ojdbc14.jar might misbehave on newer runtimes.




4. Servlet Namespace Issues (javax vs. jakarta)
The Problem:
    Your StudentServlet.java uses import javax.servlet.*;. This is perfectly correct for older versions of GlassFish (v4 or v5) and Tomcat (v9). However, if you or the lab recently updated to a very new server (like GlassFish v7 or Tomcat v10+), Oracle transitioned Java EE to Eclipse, changing the namespace to jakarta.servlet.*.

The Resolution:
    1. Since it is an older lab, javax.servlet.* is likely exactly what you need. If you get compilation errors saying "package javax.servlet does not exist," check that GlassFish is properly linked to your NetBeans project:

    2. Right-click Project -> Properties -> Libraries.

    3. Ensure Java EE Web API or the GlassFish Server library is listed. If not, add it.



5. Frontend Breakages on Old Browsers
The Problem:
    Lab computers sometimes default to outdated web browsers (like old versions of Internet Explorer or outdated Chrome versions).

    Age Calculator: You are using <input type="date">. Older browsers do not support this and will render it as a standard text box, which breaks your JavaScript date logic.

    Basic Calculator: You are using CSS display: grid; and display: flex;. Very old browsers do not support CSS Grid, so your calculator will look completely jumbled.
The Resolution:
    Always open your projects in the most updated browser available on the lab PC (prefer Chrome or Firefox over Internet Explorer). If the UI breaks completely, you might have to fall back to using HTML <table> structures instead of CSS Grid for the calculator if you are forced to use a legacy browser.
